## Overview

**What can Janet do?** is a static, self-updating portfolio that showcases Janet-related GitHub repos. It:

- Fetches repo metadata, README excerpts, and latest releases from the GitHub API
- Renders a single-page HTML site (Aero glass design, Outfit + JetBrains Mono, Double Soul section, Janet mascot)
- Auto-updates every 6 hours via GitHub Actions
- Deploys to Cloudflare Pages

**Live URLs:**

| URL | Status |
|-----|--------|
| https://heyjanet.org | Primary (DOMAIN_REGISTRY.md) — portfolio |
| https://heyjanet.org/janet/ | **Meet Janet** (mascot landing; static files in `janet/`) |
| https://what-can-janet-do-1c8.pages.dev | Direct Pages URL (this Cloudflare account; legacy `what-can-janet-do.pages.dev` may differ) |

**Dual-apex IA:** The portfolio links to **The Great Sage** (`greatsage.org`) and **Janet Glasses** (`/glasses`) in the sticky top nav and footer. `config.json` → `domain` drives `canonical` and absolute **Open Graph / Twitter** URLs; `metaDescription` fills meta/og/twitter descriptions. Hero uses **`assets/janet-mascot.png`** (ship with deploy for `og:image`). Bridge CSS: **`css/tokens.css`** (keep in sync with `greatsage-web/css/tokens.css`). Full checklist: [greatsage-web/DEPLOY_DUAL_APEX.md](../../greatsage-web/DEPLOY_DUAL_APEX.md).

**Repo:** https://github.com/MzxzD/what-can-janet-do

**Meet Janet (mascot):** The folder **`janet/`** is a copy of [janet-mascot-web](https://github.com/MzxzD/janet-mascot-web), served at **https://heyjanet.org/janet/** on the same Cloudflare Pages project (no second hostname required). After editing the mascot repo, run `./scripts/sync-janet-mascot-into-deploy.sh` from this directory, then commit `janet/` and deploy.

**Companion page (static, not generated by `build.mjs`):** [remote-access-tailscale.html](remote-access-tailscale.html) — **Great Sage** operator guide for remote Janet over Tailscale (copy-paste snippets + copy buttons). Uses the same **dual-apex sticky nav** as `index.html` (`css/tokens.css`, ecosystem links); **Remote** is `aria-current="page"` on that file. Listed from the home hero and footer. Source of truth also under **the-great-sage/janet-guides/** in the workspace.

**TRY IT YOURSELF!** (in [template.html](template.html), baked into `index.html` by `build.mjs`): a **wish grid** (base stack + optional add-ons) and **[assets/try-it-yourself.js](assets/try-it-yourself.js)** that composes **real** `brew tap` / `brew install` lines from the **MzxzD/janet** tap. Curated client-side only — **not** LLM-generated. Links to the Tailscale guide for remote clients. Branding uses **Janet cyan/blue** with **Great Sage gold** accents (shared with `remote-access-tailscale.html`).

## Architecture

```
┌─────────────────┐     ┌──────────────┐     ┌─────────────────┐
│   GitHub API    │────▶│  build.mjs   │────▶│   index.html    │
│ (repos, README, │     │  (Node.js)   │     │ (static output) │
│   releases)     │     └──────────────┘     └────────┬────────┘
└─────────────────┘            │                      │
                               │                      ▼
                     ┌─────────┴─────────┐   ┌─────────────────┐
                     │  GitHub Actions   │   │ Cloudflare Pages│
                     │ (every 6h + push) │   │  (serves site)  │
                     └───────────────────┘   └─────────────────┘
```

**Data flow:**

1. `build.mjs` reads `config.json` and fetches GitHub API for each repo
2. Renders `template.html` with placeholders, writes `index.html`
3. GitHub Actions runs on push and every 6 hours; commits `index.html` if changed
4. Cloudflare Pages deploys from the repo (static, no build step)

## Quick Start

### Run locally

```bash
cd Janet-Projects/what-can-janet-do   # or: cd ~/Documents/Janet-Projects/what-can-janet-do
node build.mjs
```

Opens `index.html` in a browser, or serve with any static server:

```bash
python3 -m http.server 8080
# or: npx serve .
```

### Deploy (Cloudflare Pages)

1. Push the repo to GitHub
2. Cloudflare Dashboard → Pages → Create project → Connect to Git
3. Select `MzxzD/what-can-janet-do`
4. Build settings: no build command, output directory `/`
5. Save and Deploy

## Configuration

Edit `config.json`:

| Key | Description |
|-----|-------------|
| `domain` | Custom domain (e.g. `heyjanet.org`) |
| `title` | Page title |
| `tagline` | Hero tagline |
| `repos` | Array of `owner/repo` to fetch |

| `footerLinks` | Links for footer |

## Deployment

Canonical runbook for **all** Cloudflare Pages properties: [WEBSITES_CLOUDFLARE_PLAYBOOK.md](../JanetOS/docs/business/WEBSITES_CLOUDFLARE_PLAYBOOK.md).

### Cloudflare Pages

- **Project:** what-can-janet-do
- **Build command:** (leave empty)
- **Build output directory:** `/`
- **Root directory:** (default)

**Recommended: manual deploy via wrangler** — Git-triggered deploys may not always update the live site. To reliably deploy into the **same Cloudflare account as the `heyjanet.org` zone** (see **DOMAIN_REGISTRY.md**):

```bash
cd ~/Documents/Janet-Projects/what-can-janet-do
# One-time if the project does not exist in this account yet:
# npx wrangler pages project create what-can-janet-do --production-branch=main
export CLOUDFLARE_ACCOUNT_ID=...   # from scripts/.env or dashboard
# API token or CLOUDFLARE_EMAIL + CLOUDFLARE_API_KEY (Global Key)
npx wrangler pages deploy . --project-name=what-can-janet-do --commit-dirty=true
```

Git push to `main` can trigger a deploy, but if the live site shows stale content, run the wrangler command above.

(Requires Node 20+ and `wrangler` CLI; `npx wrangler` works without global install.)

### `janet/` (Meet Janet on heyjanet.org)

- **URL:** `https://heyjanet.org/janet/` — bundled static assets; not produced by `build.mjs`.
- **Refresh from mascot repo:** `./scripts/sync-janet-mascot-into-deploy.sh` (expects sibling `../janet-mascot-web` or set `JANET_MASCOT_WEB`).
- **Commit:** Include changes under `janet/` when you deploy so Pages serves the updated mascot page.

### GitHub Actions

Workflow: `.github/workflows/update.yml`

- **Triggers:** Push to `main`/`master`, and cron every 6 hours
- **Steps:** Checkout → `node build.mjs` → commit & push `index.html` if changed

Uses `GITHUB_TOKEN` for higher API rate limits.

## Custom Domain (heyjanet.org)

heyjanet.org is configured in `config.json` and `CNAME`. DNS is managed at Namecheap.

**Setup checklist:** See [HEYJANET_ORG_SETUP.md](HEYJANET_ORG_SETUP.md) for a step-by-step checklist.

### Option A: Cloudflare DNS (recommended)

1. Cloudflare Dashboard → Add site → `heyjanet.org`
2. Namecheap → Domain List → Manage → Nameservers → Custom DNS
3. Enter the Cloudflare nameservers (e.g. `ada.ns.cloudflare.com`, `bob.ns.cloudflare.com`)
4. Pages → what-can-janet-do → Custom domains → Set up a custom domain → `heyjanet.org`
5. Cloudflare will create the DNS record automatically

### Option B: Namecheap DNS

1. Pages → what-can-janet-do → Custom domains → Set up a custom domain → `heyjanet.org`
2. Cloudflare shows the CNAME target (e.g. `what-can-janet-do-1c8.pages.dev`)
3. Namecheap → Advanced DNS → Add record:
   - Type: CNAME
   - Host: `@` (for apex) or `www`
   - Value: your project’s `*.pages.dev` hostname (see **DOMAIN_REGISTRY.md** / `PAGES_CNAME_TARGET` in `scripts/.env.example`)
4. For apex `@`, Namecheap may require URL Redirect Record or ALIAS; use the exact instructions Cloudflare provides

## Troubleshooting

| Issue | Solution |
|-------|----------|
| `cd Janet-Projects/what-can-janet-do` fails | Use full path: `cd ~/Documents/Janet-Projects/what-can-janet-do` or `cd ../what-can-janet-do` from janet-arm64-toolchain |
| Wrangler needs Node 20 | `nvm use 20` or upgrade Node |
| GitHub API rate limit | Set `GITHUB_TOKEN=ghp_xxx` when running `node build.mjs` |
| heyjanet.org shows parking page | DNS not yet pointed to Cloudflare Pages; follow [Custom Domain](#custom-domain-heyjanetorg) |
| index.html not updating | Check GitHub Actions tab; workflow may need `GITHUB_TOKEN` permissions |
| Live site not updating after push | Git-triggered Pages deploys can be unreliable. Run `npx wrangler pages deploy . --project-name=what-can-janet-do` for a direct deploy |
